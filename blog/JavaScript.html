JavaScript & Ruby
The Differences of Loooping

March 23rd, 2016

            
              #=> Ruby

              foods = ["pizza", "burgers", "chips"]

              foods.each do |food|
                puts food
              end

              #=> JavaScript

              var foods = ["pizza", "burgers", "chips"];

              foods.forEach(function(food) {
                console.log(food);
              });

Up there is just one way Ruby and JavaScript differ in their looping. But just take a look at the syntax for a second. Now of course you're probably thinking, eww! Whys is that JavaScript even up there? It looks so bleh! And lemme tell you, I KNOW! Ruby is definitely much more English-Friendly while JavaScript looks very loose. Ruby definitely makes it easier to loop however, getting used to the loops in JavaScript isn't so difficult. It's much easier than one might think. Let's take a look at that code piece by piece and really see what it is doing. 

              
                foods.forEach(function(food) {
                  console.log(food);
                });

The forEach being called on foods is the same as the each method being called on the same array in Ruby. But wait! Ruby has a little bit of a clearer understanding of what is going on with the code after you call the each method. In Ruby you write the keyword do followed by pipes ( || ) with a keyword to be a placeholder for each element in the array. Well JavaScript can be a little bit confusing but allow me to decipher this for you. With the forEach method, we open the parentheses. Inside we call function which is essentially the same thing as do in Ruby. I mean let's take a look at it from a logical perspective. When you tell Ruby do, you're really saying "Hey Ruby, carry out this function for each element". Well in JavaScript, instead of saying do, saying function just cuts straight to the chase. So logically, it makes sense. Then we open the parentheses for function which we now can pass in our placeholder keyword for each value in the array. Then we open the bracket signaling to Javascript that whatever is in these brackets is the block of code we want to evaluate. In this case, console.log which is the same as puts in Ruby. It will not return anything. Just simply print out to the screen, or in this case, log to the JavaScript console. And to console.log, we open parentheses and pass in what we want to log to the screen. This is fairly simple in practice. Now let me show you another bit of JavaScript code for looping and you try guessing what it does! 

              
                var names = ["Elan", "Alex", "Vlad", "Sabina", "Erica"];

                for ( var idx = 0; idx < names.length; idx++ ) {
                  console.log( names[ idx ] );
                }

The for loop can get quite confusing. But guess what, this is doing the same thing that our each method was doing a moment ago with another array. Now check this out. We say for ( var idx = 0 ). This means we are starting our for loop and setting a new variable there. The variable name can be anything and we can give it any value we like. Then we go to the next portion: for ( var idx = 0; idx < names.length ). This is setting the condition. As long as this condition is true, JavaScript will continue to loop. Then, we write our code block: 

              
                for ( var idx = 0; idx < names.length ) {
                  console.log( names[ idx ] );
                }

JavaScript will first create that variable, then it will check if the condition is true, and then it evaluates the code inside the block. What does it do after? Well, it comes back up to our last part of the "for" line like so: 

              
                for ( var idx = 0; idx < names.length; idx++ ) {
                  console.log( names[ idx ] );
                }

That last part of the "for" line gets executed last because it acts as the "update" function. We need to eventually break out of this loop. This last bit of code will update our variable that we set in the beginning after each loop through the code block until the condition is finally no longer true.

There is quite a lot more to it but for now, try to take that in. Also, it is important to know the for loop because it comes in handy when not working with arrays. Keep in mind that the forEach() method can only be called on an Array. It will not work on a string. The for loop would though! Happy looping :)
