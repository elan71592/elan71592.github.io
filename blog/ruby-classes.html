Classes. Gotta love em' huh? They're actually super awesome and can get a little confusing. However, once you get the hang of it, you'll be attached to them forever! Hopefully you got a good look at the code up there. Now let's go through it step by step. In order to declare a class method, we do: 

              
                class Person
                end

This is slightly different from a regular method that we're used to, like so: 

              
                def person
                end

Ok this is really the easy part. A class in simplest terms is like a parent and it holds all of your regular methods that you've been practicing lately in it. These methods are like it's children, or minions. I like to say minions haha. When we define a method, we define it beginning with lowercase letters. When defining a class method, we do the same thing just begin with an uppercase letter and instead of "def", we use "class". Take a look at the difference between the two.

A Class method usually starts with an initialize method. Depending on certain circumstances, this can change but for the most part, we'll stick to it. So to our Person class, we add the initialize method which will run every single time automagically when we create a new instance of this class. In case you're wondering, a new instance of this class means we are simply calling on this class to create a new object. So take a look: 

              
                class Person
                  def initialize(name, city)
                    @name = name
                    @city = city
                  end
                end

If you notice, not much here is really changing. A few things might look new but it's nothing too crazy. Let's dive in. The initialize method is being passed two arguments. Name and City. Both of which are being set to some weird looking variables. These variables with an @ sign, are called instance variables. They can be accessed within any method of the instance of that class. This sounds confusing but you'll see how cool it is in a second. When we create our next method like so: 

              
                class Person
                  def initialize(name, city)
                    @name = name
                    @city = city
                  end

                  def favorite_fruits(fruits)
                    name = @name
                    @hash = {
                      name => {favorite_fruits: fruits}
                    }
                  end
                end

Those instance variables come in handy. Now you see I can access my @name variable in my favorite_fruits method. Then I set it to a local variable name. Now remember that name is local to my favorite_fruits method so therefore I am free to use that variable to store the value of my instance variable @name. The two are NOT the same. Then I create a instance variable @hash and give it another hash inside by the name of the name variable which holds a symbol key of favorite_fruits: and a value of the fruits array passed in or it can be a string as well. Fruits being plural, I would assume as a reader that this would be an array. Anyway, I didn't have to make @hash an instance variable right? The method would have still worked if I would have just used hash. BUT, here's the thing. What if I want to know the value of hash later? How would I access it? I wouldn't if it wasn't an instance variable. I'd have to run my method again but still it wouldn't be stored somewhere I can just look at whenever I need to. So how do you use these instance variables to get information again? Simple! The attr methods. Awesome stuff we're going to look at now: 

              
                class Person
                  attr_accessor :name, :city
                  attr_reader :hash

                  def initialize(name, city)
                    @name = name
                    @city = city
                  end

                  def favorite_fruits(fruits)
                    name = @name
                    @hash = {
                      name => {favorite_fruits: fruits}
                    }
                  end
                end

Sweet huh? Did you guess what those attr methods do? Well, the accessor one might be strange but the reader one does exactly what you would think it does. It creates a method that essentially returns that instance variable @hash. Meaning, if we were to create this Person object and store it in a variable, let's say elan. All we have to do to see what @hash is, is to say elan.hash. BAM! It'll give it to you. You can also do elan.name or elan.city here and get a result for those two as well. Wait ELAN! I don't get it? The other two are attr_accessor though. Why not reader like hash is? Well that is a great question and I'm so happy you asked that hah! In Ruby, we have a reader method and a writer method. The reader returns the value. The writer method has the ability to manipulate the value. The attr_accessor method creates both automatically for you. You also have the attr_writer and attr_reader methods that you can use to create both separately for you. Now for this exercise I used accessor instead of writer because I need to use both to show you how I can read and write. So let's dive in and I think you'll get a better understanding. 

              
                class Person
                  attr_accessor :name, :city
                  attr_reader :hash

                  def initialize(name, city)
                    @name = name
                    @city = city
                  end

                  def favorite_fruits(fruits)
                    name = @name
                    @hash = {
                      name => {favorite_fruits: fruits}
                    }
                  end
                end

                # With this class method, I can create my object.

                elan = Person.new("Elan", "NYC")
                # This creates me :)

                # Then we can say:

                elan.name
                #=> "Elan"

                or:

                elan.city
                #=> "NYC"

                # We can achieve that because of the attr_methods!
                # The accessor also allows us to write though, remember?

                elan.name = "David"
                elan.name
                #=> "David"

                # Amazing right?

So there you have it! That's a short intro to classes. I hope it was somewhat understanding to you guys. Please email me if you have any questions.
