Test Driven Development

April 2nd, 2016

Test Driven Development often abbreviated as TDD is a style of software development.Kent Beck, an American software engineer is often credited with the discovery of TDD. This methodology is extremely useful in the process of software development. There are 5 key steps that a developer follows when using this process. These steps are:

Step 1: Add tests
Step 2: Run the tests to get them failing
Step 3: Code some
Step 4: Run the tests an see them pass
Step 5: Refactor

There's many reasons as to why this kind of development has become widely adopted by many software companies. For instance, when using TDD, a developer must write the tests before doing any kind of developing. This is important because it forces the developer to really understand the concepts of what the software being developed is and how the user should feel when using it. Also, by implementing TDD, developers often save a lot of time. While there is usually more code involved due to the actual tests being written, the overall time of the project can be significantly shortened because the developer has already written the tests which test for how the program should be functioning. In short, TDD is very useful, important, and most likely every developer will come across this spectacular method of programming.
