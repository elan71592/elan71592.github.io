Enumerable Methods
I'll have one of EACH!

March 12th, 2016

            
              fruits = ["apples", "pairs", "strawberries", "blueberries", "plumbs"]

              person = {
                    "name" => "elan",
                    "city" => "nyc",
                    "favorite food" => "pizza"
                    "favorite number" => 15
                    }
              # This is an array of fruits and a hash 
to store a person's attributes

Alright! Now for some fun stuff. By now I think we all know what those two types of data storage are. An array and a hash. If you don't know, check out my other Blog Post on them. Let's begin with something really awesome shall we?

One of the most common if not THE most common rubyist go-to enumerable methods, is the Each enumerable. All the way from the Gods for code, comes a method that will change your life...FOREVER! Does that sound like a good pitch for a trailer? I think there should be more movies about developers!

Ok. So what is this each method and how does it work. Well as you can most likely guess, the each method can be called on a data structure in ruby such as a hash or an array and go through each stored entry of data within them. This can make for an extremely handy tool and definitely will be used a lot in your developer life. We need to gain control over things in our data structures to manipulate them and construct programs to our specifics. If we wouldn't be able to manipulate data inside of these structures, then why have them at all right?

Now we know what each does but umm, how the heck do you use it? OK! calm down people i'll tell ya! You use it by, erm.. well here: 

              
                fruits.each do |fruit|
                  puts "I love #{fruit}!"
                end

                #=> "I love apples!" 
 #=> "I love pairs!" 
 #=> "I love strawberries!" 
 #=> "I love blueberries!" 
 #=> "I love plumbs!"

Looks pretty straight forward right? All we're doing is calling each on our fruits array by adding a ".each" to the end of fruits. Followed by that we write the word "do" which begins our loop. This is then followed by a keyword inside two pipes (ex: |fruit| ). This is telling ruby that for each item in the array, we will substitute it for this word. This in no way means we are changing each word to the new keyword. We are simply gaining control over each item in the array by using a keyword. Then your code for what you want to do with each item goes in between your do and end statement like above. Now that we got that settled. Seems easy right? Well it's no different for hashes either, except with a hash, your code would differ slightly with what goes between the pipes, like so: 

              
                person.each do |key, value|
                  puts "My #{key} is #{value}"
                end

                #=> "My name is elan" 
 #=> "My city is nyc" 
 #=> "My favorite food is pizza" 
 #=> "My favorite number is 15"

See? Simple! Arrays only need one keyword. Hashes need two. One for the key, and one for the value. In this case, my hash is one which contains attributes of myself as keys and the answers as values, so I named my keywords in that fashion but you'll come up with your own for your programs!

Now that you have an idea of the enumerable method "each". Go try it out on your own. Do something simple like favorite city. Test it out with an array of cities. Then try building a simple hash and using the each method on it. Enjoy!
